# Music Box MVP

## 1) Product Goal

Build a simple, interactive beginner piano learning site where users can:
- read one short lesson at a time,
- click keys/cards to hear notes,
- follow a clear learning path from basic solfege.

The MVP should prove one core value: **learn by reading + listening + clicking** in the same page.

## 2) Target User

- Absolute beginners in piano/music theory.
- Learners who prefer quick, visual, and interactive lessons.

## 3) MVP Scope (In)

- **Lesson navigation**
  - Left sidebar with lesson list.
  - Active lesson highlight.
  - Home route redirects to first lesson.

- **Lesson content via MDX**
  - Lesson pages rendered from `src/content/lessons/*.mdx`.
  - Dynamic route: `/lessons/[slug]`.
  - Static params generated from lesson metadata.

- **Interactive audio**
  - Clickable one-octave piano keyboard (white + black keys).
  - Clickable note cards (Do Re Mi...).
  - Audio playback with Tone.js synth.

- **Starter curriculum**
  - At least one complete lesson: `01-solfege`.
  - Covers C major basic notes and simple practice actions.

- **Basic responsive/readable UI**
  - Clean layout with sidebar + main content.
  - Usable typography and spacing for lesson reading.

## 4) Out of Scope (For Now)

- User accounts and progress sync.
- Quizzes, scoring, or gamification.
- Real piano MIDI input support.
- Multi-language system and i18n infrastructure.
- Advanced theory modules (chords, rhythm trainer, ear test).

## 5) Current Tech Baseline

- Framework: Next.js App Router
- Content: MDX
- UI: React + Tailwind CSS
- Audio: Tone.js
- Language: TypeScript

## 6) MVP Success Criteria

- User can open the app and reach a lesson in one click.
- User can click piano keys and hear correct note playback.
- User can click note cards and hear matching notes.
- Lesson content and interactive components work in the same page without errors.
- Lighthouse/basic manual check shows acceptable UX (no blocking functional issues).

## 7) Milestones

1. **Foundation**
   - Layout, sidebar, routing, MDX setup.
2. **Core interaction**
   - Tone.js playback + `PianoKeys` + `NoteCard`.
3. **First lesson delivery**
   - Publish `01-solfege` with interactive blocks.
4. **Polish**
   - Improve styles, content readability, and edge-case handling (`notFound`).

## 8) Definition of Done

- `pnpm dev` runs locally with no blocking errors.
- First lesson route is accessible and stable.
- All interactive controls trigger audible playback.
- Code is lint-clean or has only intentional, documented exceptions.
- This MVP doc and lesson content are committed together as product baseline.
